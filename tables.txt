1. users Table
Stores authentication and profile details for Candidates, Recruiters, and Admins.

Columns:
id: UUID (Primary Key, default: auto-generated)
first_name: VARCHAR(100) (Not Null)
last_name: VARCHAR(100) (Not Null)
email: VARCHAR(255) (Unique, Not Null)
phone_number: VARCHAR(30) (Nullable)
password_hash: TEXT (Not Null)
role: user_role Enum (candidate, recruiter, admin)
avatar_url: TEXT (Nullable)
company: VARCHAR(255) (Nullable)
location: VARCHAR(255) (Nullable)
created_at: TIMESTAMP (Default: current UTC time)
2. cvs Table
Holds the parsed and raw text versions of uploaded resumes.

Columns:
id: UUID (Primary Key)
user_id: UUID (Foreign Key referencing users.id with ON DELETE CASCADE)
raw_text: TEXT (Not Null)
file_url: TEXT (Nullable)
parsed_json: JSONB (Nullable - stores structured sections like education, experience)
is_active: BOOLEAN (Default: True)
created_at: TIMESTAMP (Default: current UTC time)
3. job_offers Table
Stores job descriptions published by recruiters.

Columns:
id: UUID (Primary Key)
recruiter_id: UUID (Foreign Key referencing users.id with ON DELETE CASCADE)
title: VARCHAR(255) (Not Null)
description: TEXT (Not Null)
parsed_json: JSONB (Nullable)
location: VARCHAR(255) (Nullable)
work_type: work_type Enum (remote, onsite, hybrid)
salary_min: NUMERIC
salary_max: NUMERIC
currency: VARCHAR(10) (Default: EUR / USD)
requirements: TEXT[] (Array of text requirements)
skills: TEXT[] (Array of skill tags)
experience_level: VARCHAR(100)
education_level: VARCHAR(255)
applicants_count: INTEGER (Default: 0)
status: job_status Enum (active, inactive, closed)
is_active: BOOLEAN (Default: True)
deadline: DATE
created_at: TIMESTAMP (Default: current UTC time)
4. cv_embeddings Table
Contains vector chunks of CV components for matching purposes.

Columns:
id: UUID (Primary Key)
cv_id: UUID (Foreign Key referencing cvs.id with ON DELETE CASCADE)
embedding_type: embedding_type_enum Enum (skills, experience, projects, title, summary)
chunk_index: INTEGER (Not Null)
chunk_text: TEXT (Not Null)
embedding: VECTOR(768) (Not Null, generated by Ollama/OpenAI)
metadata: JSONB
created_at: TIMESTAMP
5. job_embeddings Table
Contains vector chunks of job offers for matching purposes.

Columns:
id: UUID (Primary Key)
job_offer_id: UUID (Foreign Key referencing job_offers.id with ON DELETE CASCADE)
embedding_type: embedding_type_enum Enum (skills, experience, projects, title, summary)
chunk_index: INTEGER (Not Null)
chunk_text: TEXT (Not Null)
embedding: VECTOR(768) (Not Null)
metadata: JSONB
created_at: TIMESTAMP
6. applications Table
Links candidates to job offers and tracks their status.

Columns:
id: UUID (Primary Key)
candidate_id: UUID (Foreign Key referencing users.id with ON DELETE CASCADE)
job_offer_id: UUID (Foreign Key referencing job_offers.id with ON DELETE CASCADE)
cv_id: UUID (Foreign Key referencing cvs.id with ON DELETE SET NULL)
status: VARCHAR(50) (Default: 'pending')
cover_letter: TEXT (Nullable)
recruiter_notes: TEXT (Nullable)
created_at: TIMESTAMP
updated_at: TIMESTAMP
7. Chat History Tables (chat_sessions & chat_messages)
Stores message histories for the integrated AI assistant.

chat_sessions Table:
id: UUID (Primary Key)
user_id: UUID (Foreign Key referencing users.id with ON DELETE CASCADE)
created_at: TIMESTAMP
chat_messages Table:
id: UUID (Primary Key)
session_id: UUID (Foreign Key referencing chat_sessions.id with ON DELETE CASCADE)
role: message_role Enum (user, assistant, system)
message: TEXT (Not Null)
created_at: TIMESTAMP
